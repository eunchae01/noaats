다음 자바 파일에 대한 단위 테스트 작성 : $ARGUMENTS

요구사항:
- JUnit 5 프레임워크 사용
- BDD 스타일의 테스트 메서드 이름 사용 (예: givenX_whenY_thenZ)
- BDDMockito를 사용하여 의존성 모킹
- 주요 시나리오에 대한 테스트 커버리지 포함
- 테스트 클래스는 동일한 패키지 구조를 따름
- 테스트 클래스는 `src/test/java` 디렉터리에 위치
- 테스트 클래스는 `@ExtendWith(MockitoExtension.class)`로 주석 처리
- 각 테스트는 독립적이어야 하며, 공통 설정은 `@BeforeEach` 메서드에서 처리
- AssertJ를 사용하여 가독성 높은 단언문 작성
- 테스트 클래스는 `*Test` 접미사를 가짐
- 테스트 메서드는 `@Test`로 주석 처리
- 필요한 경우 `@Mock` 및 `@InjectMocks`를 사용하여 의존성 주입
- 예외 상황에 대한 테스트 포함
- 테스트 데이터는 테스트 메서드 내에서 직접 정의
- 테스트 실행 시 외부 시스템에 의존하지 않음 (예: 데이터베이스, 파일 시스템)
- 테스트 결과는 일관되고 재현 가능해야 함
- test/java/docs 하위에 동 이름으로 같은 경로를 가진 디렉토리에 md 파일 생성
