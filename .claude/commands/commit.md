다음 행위에 관련된 파일 포함하여 커밋 작성 : $ARGUMENTS

요구사항:
- 커밋 메시지는 명확하고 간결해야 함
- 커밋 메시지는 변경 사항의 목적을 설명해야 함
- 커밋 메시지는 현재 시제로 작성 (예: "Fix bug", "Add feature")
- 커멧 메시지는 한글로 작성
- 같은 행위를 한 파일은 모두 하나의 커밋에 포함하여 작성
- 단 동일 행위에 대한 항목이 많을 경우 세부항목으로 구분하여 별도커밋 진행
- 커밋 당 변경 파일 수는 10개 이하로 제한
- 커밋 메시지 앞에는 [YYYY-MM-DD] 형식의 날짜를 포함
- 커밋 메시지는 헤더에만 작성, body, footer는 작성하지 않음
