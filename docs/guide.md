1. 과제 개요

2. 제출물 구성
   지원자는 아래 2가지 제출물을 반드시 제출해야 합니다. 둘 중 하나라도 누락될 경우 평가 대상에서 제외됩니다. ▲ 기획서 및 개발 문서 ▲ 동작하는 웹앱 프로토타입
   2.1 기획서 및 개발 문서
   과제 수행 과정과 사고의 흐름을 보여주는 문서입니다. 형식은 자유이며(PDF, Markdown, Notion 링크 등), 아래 항목을 반드시 포함하여야 합니다.
   (1) 문제 정의 (가장 중요)
    해결하려는 문제가 무엇인지 구체적으로 설명
    왜 이 문제를 선택했는지에 대한 근거 제시
    핵심 기능과 부가 기능을 구분하여 정리
    대상 사용자 정의 및 사용자 시나리오 작성
    예상되는 예외 상황(엣지 케이스) 및 처리 방안
   **안내** 주제는 방향성만 제시합니다. 예를 들어 “대출 관리”라는 주제가 주어졌다면, 그 안에서 “구체적으로 어떤 문제를 풀 것인지는 지원자가 직접 정의”해야 합니다. 문제 정의 역량은 전체 평가에서 가장 큰 비중(40점)을 차지합니다.
   (2) 개선 과정 기록
    처음 설계와 최종 결과물 사이의 변경 사항
    문제를 어떻게 단계적으로 해결해 나갔는지
    막혔던 부분과 그 해결 방법
    시간 배분과 우선순위 조정 내역
   (3) AI 활용 기록 (권장)
    AI에게 어떤 컨텍스트(배경/요구사항/제약 등)를 제공했는지
    AI 생성 결과물을 어떻게 검증하고 수정했는지
    AI와의 협업에서 발견한 문제점과 대응 방식
   2.2 동작하는 웹앱 프로토타입
    실제로 동작하는 웹 애플리케이션
    배포 URL 또는 로컬 실행 가능한 소스코드(GitHub 저장소 등)
    소스코드에는 README 파일 포함 필수(실행 방법 명시)
   **안내** 배포까지 완료하면 가산점이 있습니다.(Vercel, Netlify, GitHub Pages 등)

7. 자주 묻는 질문(FAQ)
   Q1. AI 도구를 사용하면 감점되나요? A1. 아닙니다. AI 사용은 오히려 권장됩니다. AI를 활용하여 얼마나 좋은 결과물을 만들어내는지가 핵심입니다.
   Q2. 기술 스택은 자유인가요? A2. 네. 언어, 프레임워크, 데이터베이스 등 모든 기술 선택은 자유입니다.
   Q3. 완성도가 중요한가요, 사고 과정이 중요한가요? A3. 사고 과정(문제 정의, 개선 과정)이 완성도보다 더 높은 비중을 차지합니다. 문제 정의 능력(40점)이 전체 배점의 40%입니다.
   Q4. 반드시 웹앱이어야 하나요? A4. 네. 동작하는 웹 애플리케이션 프로토타입을 제출해야 합니다.
   Q5. 디자인이 예뻐야 하나요? A5. 디자인은 가산점(+1점) 항목입니다. 기능 정확성과 문제 정의가 훨씬 중요합니다.
   Q6. Git 커밋은 반드시 있어야 하나요? A6. 필수는 아니지만, 개선 과정 기록(8점)에서 Git 커밋 히스토리가 가장 명확한 증거가 됩니다. 기획서에 개선 과정을 서술하는 것도 가능합니다.
